# 🛍️ 电商评价情感洞察与选品策略 Agent

基于 **LangGraph** 构建的多 Agent 协作系统，自动对商品评论进行细粒度属性级情感分析，结合退货率等业务数据执行长链推理，生成选品/改进报告，并交叉验证社交媒体舆情。

> 该项目展示了一种将 LLM 应用于电商“评价分析→供应链归因→决策建议”完整链路的可行方案，适合作为 AI Agent 落地的原型或面试作品。

---

## 📌 项目背景与核心痛点

- **海量评论**：运营团队每天面对数万条用户评价，人工只能抽查 <5% 的样本。
- **情感粒度粗**：传统情感分析仅分正面/负面，无法定位是“掉色”“起球”还是“尺码不准”导致了差评。
- **归因链条长**：从一条“起球”差评推断出可能是纱线捻度不足、面料等级偏低，需要跨领域（供应链）推理，人工效率极低。
- **舆情滞后**：当某个质量缺陷在社交平台发酵后，品牌方才紧急处理，错过最佳干预窗口。

本系统通过四个专职 Agent 协同工作，将**响应周期从数周压缩至分钟级**，实现“评价→洞察→行动”的自动化闭环。

## 🧠 系统架构（多 Agent 协作流）
用户评论 & 业务数据
│
▼
┌─────────────────┐
│ Crawler Agent │ ← 采集评论、退货率、复购率等
└───────┬─────────┘
│ 原始评论 + 基础数据
▼
┌─────────────────┐
│ NLP Agent │ ← 属性级情感抽取（属性、情感、证据）
└───────┬─────────┘
│ 属性情感列表
▼
┌─────────────────┐
│ Insight Agent │ ← 长链推理：高频负面词 → 供应链缺陷 → 影响评估
└───────┬─────────┘
│ 初步洞察
▼
┌─────────────────┐
│ Trend Agent │ ← 社交平台热词交叉验证，确认外部舆情强度
└───────┬─────────┘
│ 交叉验证结论
▼
┌─────────────────┐
│ Reporter Agent │ ← 汇总生成结构化选品/改进报告
└─────────────────┘

text

整个流程由 **LangGraph StateGraph** 编排，状态在节点间自动流转，支持断点调试与记忆保持。

## ✨ 核心特性

- **属性级情感挖掘**：不只是“好评/差评”，而是抽取出 `[起球, 负面, "穿两天就起球"]` 这样的细粒度三元组。
- **长链推理引擎**：从表面差评推断出可能的供应链问题（如色牢度、纱线品质），并关联退货率、复购率评估严重性。
- **社交媒体交叉验证**：模拟抓取微博、小红书相关热词，判断是否有外部舆情发酵风险。
- **结构化报告自动生成**：最后输出一份包含“问题定位-供应链根因-舆情强度-行动建议”的四段式决策报告。
- **低耦合可扩展**：每个 Agent 独立封装，可单独替换为真实 API 或更复杂的模型。
